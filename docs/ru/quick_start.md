# Быстрый старт или как вообще тут работать

Для начала, стоит ознакомиться с `.env` файлом - на данный момент это основная точка сбора всех переменных, секретов и всякого такого. Возможно, позже это будет в отдельных конфигах, но пока только так.

Разберёмся по порядку.

Вот пример "конфига": [.env](../../.env.example)

## Переменные окружения, чё да как

### Важно!
- `MARIONETTE_TOKEN` - это буквально сердце нашего презентера, то бишь Discord-бота. Заходим на [DDevs портал](https://discord.com/developers/applications), создаём своё приложение, берём оттуда токен и вставляем.

- `DATABASE_URL`, `DATABASE_POOL_SIZE`, `DATABASE_MAX_OVERFLOW`, `DATA_SOURCE_NAME`, `REDIS_URL` - это всё, в целом, сильно в настройке не нуждается, в примере уже ready-to-go переменные, собранные под конкретно этот проект, Docker сам из вашего `.env` файла подхватит их и всё будет работать :)

- `MAIN_GUILD_ID` - бот конструировался под локальное комьюнити, так что заточен он под один сервер, а не публику, так что вам придётся выбрать один сервер, на котором бот будет работать

### Уже не очень важно...
Все переменные ниже - как бы нужны, но как бы и нет, потому что от них зависит некоторый функционал, но в целом бот запустится и без них. Если вам не нужен функционал бота, то вы можете попросту вырезать эти переменные и вырезать соответствующий функционал отсюда.

## Почему RP_CATEGORIES такой необычный?
Потому что в проекте используется библиотека `dature` и она позволяет читать подобные переменные как python-списки, из-за чего получается такой синтаксис.

# Makefile
Итак, мы разобрались с переменными, идём дальше. Как запускать? Вероятно, вы видели в `README` команду `make up`, однако там есть и места поинтереснее.

## Старт и стоп
Для старта:

```bash
make up
```

Для остановки:
```bash
make down
```

## Что ещё?
Помимо всего прочего есть `make restart` - перезапуск приложения и мониторинга.

## Мониторинг?
Да, при вызове `make up` поднимается `Victoria Metrics` с `Grafana`, которую можно будет настроить для лучших метрик.

**Также**, можно отдельно запустить мониторинг: `make monitoring-up` и по аналогии выключить `make monitoring-down`.

## Логи
Для отслеживания логов в контейнере, можно использовать `make logs` - автоматически подключит вас к логам контейнера.

## Очистка
Если вам мозолит глаза `__pycache__` - его можно будет удалить незамысловатой `make clear`

# А если без докера?
Ну тогда всё сложнее. Вам придётся грузить переменные `.env` в своё окружение, либо использовать `python-dotenv`, что не очень чистая и хорошая практикаю.

И забудьте в таком случае про `make` - он работает лишь с Docker (исключение: `make clear`)

Вам нужно: uv, python 3.14.

Пишете: `uv pip install -e .` - это установит Marionette как отдельный пакет.

А далее `uv sync`, `python -m marionette`, но перед этим вам также локально придётся поднять хотя бы Postgres, чего я объяснять уже не буду.
